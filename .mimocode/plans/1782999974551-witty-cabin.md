# Fix: Windows Game UI Preview Stuck on First Frame

## Root Cause

The bug is in `apps/control-center/src/operator/center/HeroFrame.tsx` (line 64). The `useEffect` that fetches the screenshot for Windows adapter has an incomplete dependency array:

```javascript
}, [sessionId, adapterType]);  // <-- Only runs when sessionId or adapterType changes!
```

This means the screenshot is fetched **ONCE** when the session starts, and never re-fetched even though:
- The session is polled every 3 seconds (in `useSessionPolling`)
- Steps and events are updated
- New screenshots are available on the backend

**Contrast with `WindowsRpaPanel.tsx`** (line 62) which correctly polls:
```javascript
load();
const id = setInterval(load, 2000);  // Polls every 2 seconds
```

## Files to Change

1. **`apps/control-center/src/operator/center/HeroFrame.tsx`** — Add polling mechanism
2. **`apps/control-center/src/operator/center/HeroFrame.test.ts`** — Add regression test (new file)

## Fix

Add a polling interval to `HeroFrame.tsx` for Windows adapter, matching the pattern in `WindowsRpaPanel.tsx`:

```typescript
useEffect(() => {
  if (adapterType !== "windows" || !sessionId) {
    setDataUrl(null);
    return;
  }

  let cancelled = false;
  
  const fetchScreenshot = async () => {
    try {
      const r = await fetch(`http://127.0.0.1:8100/sessions/${sessionId}/screenshot`);
      if (!r.ok) throw new Error(`HTTP ${r.status}`);
      const data = await r.json();
      if (cancelled) return;
      if (data.data_base64) {
        setDataUrl(`data:image/png;base64,${data.data_base64}`);
      } else {
        setDataUrl(null);
      }
    } catch {
      if (!cancelled) {
        setDataUrl(null);
      }
    }
  };

  fetchScreenshot();
  const id = setInterval(fetchScreenshot, 2000);
  
  return () => {
    cancelled = true;
    clearInterval(id);
  };
}, [sessionId, adapterType]);
```

**Key changes:**
1. Extract fetch logic into `fetchScreenshot` function
2. Call `fetchScreenshot()` immediately
3. Add `setInterval(fetchScreenshot, 2000)` for polling
4. Cleanup interval in return function
5. Remove `setFetching` state (not needed for polling - only show loading on initial fetch)
